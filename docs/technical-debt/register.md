# 기술부채 등록부

## 유형 분류

| 유형 | 설명 | 등록 시점 |
|---|---|---|
| `code-debt` | 코드 품질/구조 부채 (리팩토링 필요, 우회 구현 등) | 구현/리뷰 단계 |
| `pending-confirm` | 고객사 확인 대기 또는 변경 가능성 높은 요구사항 | 요구사항 분석 단계 |
| `assumption` | 정보 부족으로 가정 기반 구현한 항목 | 구현 단계 |
| `external-dep` | 외부 시스템/API 스펙 미확정 | 기술설계 단계 |

## 운영 규칙

- 요구사항 분석 시 불확실 항목이 발견되면 `pending-confirm`으로 즉시 등록합니다.
- 구현 코드에 `// TD-xxx: 유형` 주석을 남겨 코드와 부채를 연결합니다.
- `pending-confirm`, `assumption` 항목은 고객사 확인 후 해소하거나 `code-debt`로 전환합니다.
- 해소된 항목은 행을 삭제하지 않고 상환 계획 컬럼에 해소 일자와 결과를 기록합니다.
