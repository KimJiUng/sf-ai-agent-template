# 오케스트레이션 가이드

## 목적

Claude가 요구사항 수신부터 Org 반영까지 일관된 절차를 따르도록 합니다.

## 오케스트레이션 흐름

```text
요구사항 명확화 → 기술설계 → 디자인(필요시) → 구현 → 자체리뷰 → Org 반영
```

## 단계별 상세

### 1. 요구사항 명확화

- 사용자의 요구사항을 받으면 하네스 체크리스트를 점검합니다.
- 불분명한 항목이 있으면 구현 전에 질문합니다.
- 확정된 요구사항을 `context/open_tasks.md`에 기록합니다.

하네스 체크리스트:

1. 비즈니스 목적이 명확한가?
2. 대상 오브젝트/필드가 특정되었는가?
3. UI 요구사항이 있는 경우 레이아웃/동작 기준이 있는가?
4. 비즈니스 규칙(조건/예외/권한)이 정의되었는가?
5. 완료 조건(인수 기준)을 판단할 수 있는가?

### 2. 기술설계

- 변경 영향 분석을 수행합니다.
- 설계 의사결정을 `context/decisions.md`에 기록합니다.
- 테스트 전략과 롤백 가능성을 명시합니다.

### 3. 디자인 (필요시)

- SLDS 표준을 기본으로 사용합니다.
- 산출물은 `docs/design/specs/`에 작성합니다.

### 4. 구현

- `docs/architecture.md`의 계층 원칙을 준수합니다.
- 보안/품질 규칙(CRUD/FLS, Bulkify)을 준수합니다.

### 5. 자체리뷰

- CRUD/FLS 준수 여부
- Bulkify 기준 준수
- 하드코딩된 ID/값 없음
- 테스트 코드 포함 여부
- 기존 코드와의 충돌 가능성

### 6. Org 반영

- Deploy Gate를 실행합니다.
- 위반이 있으면 **배포하지 않고** 사용자에게 보고합니다.
- 위반이 없으면 배포를 진행합니다.

## 단계 전환 규칙

- 다음 단계는 이전 단계 필수 산출물이 준비된 경우에만 시작합니다.
- 단계 생략/되돌림은 사유를 `context/decisions.md`에 기록합니다.
